Projects Guide

Welcome to the Projects Guide, your step-by-step manual for managing components within your Angular application. Follow these instructions to seamlessly create, integrate, and manage your project components.

Creating a New Component

1. Navigate to Components: In your project directory, go to components -> projects.

2. Open Integrated Terminal: Right-click and select "Open in Integrated Terminal".

3. Generate Component: Use the Angular CLI to generate a new component with the command:

{{ generateComponentCode }}

Replace <component-name> with your desired component's name.

Deleting Unnecessary Files

To keep your project clean, you may want to delete test files. Locate any .spec.ts files within your component directory and delete them if they are not needed.

Linking Component as a New Project

1. Create Component Object: In projects-showcase.component.ts, define a new component object. For example:

{{ componentObjectCode }}

2. Define a Route: In the app.routes.ts file, create a sub-route for your project. Ensure that your project link starts with project/ followed by the project ID. Here's how you can define a route:

{{ projectPathCode }}

Replace [id] with your project's ID and [YourProjectComponent] with the actual component class.

Resolving Component References

If your component name is highlighted in red, the project does not recognize the component. In this case, you need to import it at the top of your file. Refer to the import statements of other components for syntax guidance.

Remember, import statements are always placed at the top of a TypeScript file.
